# Bytecode Instruction Set (OpCodes)

This document defines the current instruction set for the virtual machine. Each instruction is represented by a 1-byte OpCode.

| OpCode (Dec) | OpCode (Hex) | Mnemonic                  | Operands                    | Stack Effect                      | Description                                                                                                                                      |
| :----------- | :----------- | :------------------------ | :-------------------------- | :-------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------- |
| **1**        | `0x01`       | `ADD`                     | None                        | `[b, a] -> [b + a]`               | Pops two values, adds them, and pushes the result.                                                                                               |
| **2**        | `0x02`       | `SUB`                     | None                        | `[b, a] -> [b - a]`               | Pops two values, subtracts `a` from `b`, and pushes the result.                                                                                |
| **3**        | `0x03`       | `MUL`                     | None                        | `[b, a] -> [b * a]`               | Pops two values, multiplies them, and pushes the result.                                                                                         |
| **4**        | `0x04`       | `DIV`                     | None                        | `[b, a] -> [b / a]`               | Pops two values, divides `b` by `a`, and pushes the result.                                                                                      |
| **5**        | `0x05`       | `MOD`                     | None                        | `[b, a] -> [b % a]`               | Pops two values, computes `b % a`, and pushes the result. *(Note: Code generation is implemented, but execution in VM is currently missing).*   |
| **6**        | `0x06`       | `LOAD_CONST`              | 1: `cpIdx`                  | `[] -> [value]`                   | Loads a constant from the Constant Pool at index `cpIdx` and pushes it.                                                                          |
| **7**        | `0x07`       | `NEGATE`                  | None                        | `[value] -> [-value]`             | Pops a value and pushes its numeric negation.                                                                                                    |
| **8**        | `0x08`       | `NOT`                     | None                        | `[value] -> [!value]`             | Pops a value and pushes its logical negation.                                                                                                    |
| **9**        | `0x09`       | `LOAD_NULL`               | None                        | `[] -> [null]`                    | Pushes `null` onto the stack.                                                                                                                    |
| **10**       | `0x0A`       | `DEFINE_VAR`              | 1: `nameIdx`                | `[value] -> []`                   | Pops a value and defines it in the Symbol Table.                                                                                                 |
| **11**       | `0x0B`       | `LOAD_VAR`                | 1: `nameIdx`                | `[] -> [value]`                   | Retrieves the value of a variable and pushes it.                                                                                                 |
| **12**       | `0x0C`       | `UPDATE_VAR`              | 1: `nameIdx`                | `[value] -> [value]`              | Updates a variable with the popped value and pushes the value back.                                                                              |
| **13**       | `0x0D`       | `JUMP_IF_FALSE`           | 1: `target`                 | `[cond] -> []`                    | Pops `cond`. If `false`, jumps to `target`.                                                                                                      |
| **14**       | `0x0E`       | `JUMP`                    | 1: `target`                 | `[] -> []`                        | Unconditionally jumps to `target`.                                                                                                               |
| **15**       | `0x0F`       | `GREATER_THAN`            | None                        | `[b, a] -> [a > b]`               | Pops `b`, then `a`. Pushes `true` if `a > b`.                                                                                                    |
| **16**       | `0x10`       | `LESS_THAN`               | None                        | `[b, a] -> [a < b]`               | Pops `b`, then `a`. Pushes `true` if `a < b`.                                                                                                    |
| **17**       | `0x11`       | `GREATER_THAN_OR_EQUAL`   | None                        | `[b, a] -> [a >= b]`              | Pops `b`, then `a`. Pushes `true` if `a >= b`.                                                                                                   |
| **18**       | `0x12`       | `LESS_THAN_OR_EQUAL`      | None                        | `[b, a] -> [a <= b]`              | Pops `b`, then `a`. Pushes `true` if `a <= b`.                                                                                                   |
| **19**       | `0x13`       | `EQUAL`                   | None                        | `[b, a] -> [a == b]`              | Pops `b`, then `a`. Pushes `true` if `a === b` (strict equality).                                                                                |
| **20**       | `0x14`       | `NOT_EQUAL`               | None                        | `[b, a] -> [a != b]`              | Pops `b`, then `a`. Pushes `true` if `a !== b` (strict inequality).                                                                              |
| **21**       | `0x15`       | `ENTER_SCOPE`             | None                        | `[] -> []`                        | Enters a new block scope by pushing a new Map onto the scope stack.                                                                              |
| **22**       | `0x16`       | `EXIT_SCOPE`              | None                        | `[] -> []`                        | Exits the current block scope by popping it from the scope stack.                                                                                |
| **23**       | `0x17`       | `POP`                     | None                        | `[value] -> []`                   | Pops the top value off the evaluation stack.                                                                                                     |
| **24**       | `0x18`       | `DEFINE_CONST`            | 1: `nameIdx`                | `[value] -> []`                   | Pops a value and defines it as a constant in the Symbol Table.                                                                                   |
| **25**       | `0x19`       | `JUMP_IF_TRUE`            | 1: `target`                 | `[cond] -> []`                    | Pops `cond`. If `true`, jumps to `target`.                                                                                                       |
| **26**       | `0x1A`       | `DUP`                     | None                        | `[value] -> [value, value]`       | Duplicates the top value on the stack.                                                                                                           |
| **27**       | `0x1B`       | `CALL_NATIVE`             | 1: `nameIdx`, 2: `arity`    | `[arg1, ..., argN] -> [result]`   | Pops `arity` arguments, retrieves native function name at `nameIdx` from Constant Pool, executes the native function, and pushes the result.     |
| **28**       | `0x1C`       | `CALL`                    | 1: `fnIdx`, 2: `arity`      | `[arg1, ..., argN] -> [result]`   | Calls a user-defined function with `arity` arguments, pushing the return address to the call stack and jumping to the entry point.               |
| **29**       | `0x1D`       | `DEFINE_FN`               | 1: `fnNameIdx`, 2: `arity`  | `[] -> []`                        | Defines a user-defined function in the Symbol Table with `arity` and the entry point set to the current instruction pointer.                     |
| **30**       | `0x1E`       | `RETURN`                  | None                        | `[] -> []`                        | Returns from a user-defined function, restoring control flow to the caller's return address and popping the scope.                                |
| **31**       | `0x1F`       | `AND`                     | None                        | `[b, a] -> [b & a]`               | Pops two values, computes logical AND, and pushes the result.                                                                                    |
| **32**       | `0x20`       | `OR`                      | None                        | `[b, a] -> [b &#124; a]`          | Pops two values, computes logical OR, and pushes the result.                                                                                     |
