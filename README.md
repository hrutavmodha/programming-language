# Custom Bytecode Compiler & Virtual Machine (Toy Programming Language)

This repository contains a full-stack, typescript-based compiler and virtual machine implementation for a custom block-scoped scripting language. The pipeline includes:
1. **Lexical Analysis (Lexer)**: Tokenization of source text into discrete syntactic units.
2. **Syntactic Analysis (Parser)**: Recursive descent parsing to build an Abstract Syntax Tree (AST).
3. **Semantic Analysis (Analyzer)**: Scoping rules verification, constant re-assignment protection, and syntax restrictions (e.g., `break`/`continue` validation).
4. **Code Generation (Generator)**: Compiles the AST into custom bytecode instructions and populates the Constant Pool.
5. **Execution (Virtual Machine)**: A stack-based executor that runs the generated bytecode.

---

## Architecture & Compiler Pipeline

The project is structured as follows:

```
src/
├── lexer/             # Lexical analyzer & token rules
├── parser/            # Recursive descent parser producing AST
├── analyzer/          # Semantic AST analysis & validation
├── generator/         # Bytecode compiler producing VM instructions
├── executor/          # Stack-based Virtual Machine execution engine
├── stdlib/            # Standard library interface (I/O, etc.)
└── shared/            # Shared data structures (Constant Pool, Scope, etc.)
```

### 1. Lexer
The lexer ([src/lexer/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/lexer/index.ts)) scans the source code, ignoring whitespaces and comments.
- **Single-line Comments**: Start with the `#` symbol.
- **Multi-line Comments**: Enclosed in `/*` and `*/`.
- **Keywords**: `let`, `const`, `if`, `else`, `while`, `do`, `for`, `switch`, `case`, `default`, `function`, `return`, `break`, `continue`.
- **Literals**: Numbers (integers and floating-point), Strings (delimited by `'` or `"`), and Booleans (`true`/`false`).

### 2. Parser
The parser ([src/parser/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/parser/index.ts)) utilizes recursive descent to construct AST nodes. Semicolons are required after variable declarations, constant declarations, return statements, and expression statements. Semicolons are not used after blocks (like `if`, `while`, `function` bodies).

### 3. Semantic Analyzer
The analyzer ([src/analyzer/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/analyzer/index.ts)) walks the AST to enforce safety rules prior to bytecode generation:
- Checks for undeclared or duplicate variable declarations in the same scope.
- Prevents re-assignment of constant variables.
- Ensures `break` and `continue` statements are strictly placed inside loops.
- Asserts `return` statements are only used inside functions.
- Validates function call arity matching their declaration parameters.

### 4. Bytecode Generator
The generator ([src/generator/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/generator/index.ts)) compiles the validated AST into a linear sequence of instruction bytes. It registers all primitive literals (Numbers, Strings, Booleans) and variable/function names in a `Constant Pool`, referencing them in instructions using index operands.

### 5. VM Executor
The virtual machine ([src/executor/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/executor/index.ts)) is a stack-based executor implementing a 1-byte opcode instruction cycle. The VM uses an evaluation stack for expressions and a call stack to handle user-defined functions. Details on every bytecode instruction can be found in [OPCODES.md](file:///home/hrutav-modha/Documents/programming-language/OPCODES.md).

## Language Syntax & Features

### Variables & Constants
Variables are block-scoped and declared using `let`, while constants are declared using `const`. Constants must be initialized when declared.
```javascript
let count = 1;
const TAX_RATE = 0.05;
```

### Control Flow
- **If-Else Conditional**:
  ```javascript
  if x > 10 {
      print("x is large");
  } else {
      print("x is small");
  }
  ```
- **While Loop**:
  ```javascript
  while count <= 5 {
      print(count);
      count = count + 1;
  }
  ```
- **Do-While Loop**:
  ```javascript
  do {
      count = count - 1;
  } while count > 0;
  ```
- **For Loop**:
  ```javascript
  for let i = 0; i < 10; i = i + 1 {
      print(i);
  }
  ```
- **Switch Case**:
  ```javascript
  switch status {
      case 1 {
          print("Status is OK");
      }
      case 2 {
          print("Status is Pending");
      }
      default {
          print("Status is Unknown");
      }
  }
  ```

### Functions & Scope
Functions are declared using the `function` keyword, taking zero or more arguments, and optionally returning a value.
```javascript
function greet(user) {
    print("Welcome, " + user);
    return true;
}
```

### Built-in Native Functions
- `print(value)`: Prints the representation of `value` followed by a newline.
- `input(prompt)`: Displays the `prompt` string and waits for user input from stdin, returning the input line as a string.

## Complete Programming Example

Below is a complete code snippet written in the language demonstrating function declaration, parameter passing, mathematical expressions, variable scoping, and conditional evaluation:

```javascript
# Define a helper function to calculate total cost with tax
function calculateTotalCost(price, quantity, taxRate) {
    let subtotal = price * quantity;
    let taxAmount = subtotal * taxRate;
    let total = subtotal + taxAmount;
    return total;
}

# Define constant configuration
const STATE_TAX = 0.08;

# Declare variables
let itemPrice = 12.5;
let itemQty = 4;

# Execute the function and capture the return value
let finalBill = calculateTotalCost(itemPrice, itemQty, STATE_TAX);

# Check the final result and print messages accordingly
if finalBill > 50 {
    print("Over Budget! Total cost: " + finalBill);
} else {
    print("Under Budget. Total cost: " + finalBill);
}
```

## Getting Started

### Installation
Ensure you have [Node.js](https://nodejs.org) installed, then install the package dependencies:
```bash
npm install
```

The compiler pipeline entry point is located in [src/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/index.ts). You can execute the code using `tsx` or another TypeScript execution tool:
```bash
npx tsx src/index.ts
```
