# HM Custom Bytecode VM Programming Language

A custom, compiled, and interpreted stack-based programming language featuring a full compiler pipeline, intermediate bytecode generation, and a high-performance stack-based virtual machine execution environment.

---

## Architecture Overview

The system compiles source scripts down to custom bytecode, which is then verified and executed on a virtual machine. The implementation pipeline consists of five key phases:

1. **Lexical Analysis (Lexer)**: Operates within [lexer/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/lexer/index.ts). It splits the input source character stream into a sequence of tokens based on syntax rules.
2. **Syntactic Analysis (Parser)**: Defined in [parser/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/parser/index.ts). It consumes the token stream and parses it into a strongly structured Abstract Syntax Tree (AST).
3. **Static Analysis & Type Verification (Analyzer)**: Implemented in [analyzer/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/analyzer/index.ts). Resolves scopes, tracks loops and function boundaries, checks parameter counts (arity), and verifies basic type compatibility.
4. **Bytecode Generation (Generator)**: Located in [generator/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/generator/index.ts). Converts the verified AST into compact, one-byte instruction sequences, storing literals and identifier names in a centralized Constant Pool.
5. **Bytecode Virtual Machine (Executor)**: Found in [executor/index.ts](file:///home/hrutav-modha/Documents/programming-language/src/executor/index.ts). Runs the generated bytecode using a stack-based machine, maintaining evaluation and call stacks.

The main entry point bridging all components is [index.ts](file:///home/hrutav-modha/Documents/programming-language/src/index.ts).

## Language Reference & Syntax

### 1. Variables and Constants

Variables and constants are declared using block scope semantics:

- **Variables** (`let`): Can be optionally initialized at declaration and mutated later.
  ```
  let x = 10;
  let y; // Defaults to null
  x = 20;
  ```
- **Constants** (`const`): Must be initialized at declaration and cannot be reassigned.
  ```
  const PI = 3.14159;
  ```

### 2. Operators

- **Arithmetic**: `+` (addition/concatenation), `-` (subtraction/negation), `*` (multiplication), `/` (division), `%` (modulo).
- **Comparison**: `==` (equality), `!=` (inequality), `>` (greater than), `<` (less than), `>=` (greater or equal), `<=` (less or equal).
- **Logical**: `&` (logical AND), `|` (logical OR), `!` (logical NOT). Note that single characters are used for logical operations instead of double characters (`&&`, `||`).

### 3. Control Flow

#### If-Else Conditions
Parentheses around conditions are optional. Statement bodies are typical block statements:
```
if x > 10 {
    print("Greater");
} else {
    print("Smaller or equal");
}
```

#### Loops
- **While Loop**:
  ```
  while x > 0 {
      x = x - 1;
  }
  ```
- **Do-While Loop**:
  ```
  do {
      x = x + 1;
  } while x < 10;
  ```
- **For Loop**: Initialized using a `let` statement containing a trailing semicolon:
  ```
  for let i = 0; i < 5; i = i + 1 {
      print(i);
  }
  ```
- **Break & Continue**: Standard loop iteration control using `break;` and `continue;`.

#### Switch Case
A custom block format which does **not** use colons (`:`) after case labels:
```
switch expression {
    case 1 {
        print("One");
    }
    case 2 {
        print("Two");
    }
    default {
        print("Default branch");
    }
}
```

### 4. Functions
Declared using the `function` keyword, accepting zero or more arguments and returning a value:
```
function add(a, b) {
    return a + b;
}

let result = add(5, 7);
```

### 5. Classes and Object-Oriented Programming (OOP)
Supports classes with properties, methods, constructor initialization, and single inheritance.

- **Declaration**: Declared using the `class` keyword.
- **Properties**: Declared inside the class block with initial values and optional accessibility/static modifiers (`public`, `private`, `static`). Semicolons are required.
- **Initialization**: An optional `init` method acts as the instance constructor.
- **Inheritance**: Defined using the `inherits` keyword.
- **Member Access & Visibility Checks**: The static analyzer enforces access control. Specifically:
  - Private methods and properties (marked `private`) cannot be accessed outside the class scope (i.e., on class instances).
  - Static methods and properties (marked `static`) cannot be accessed on instances of the class.
  - Illegal access triggers an analyzer error: `Cannot access static or private member of class on instance of class`.

```
class Animal {
    public name = "Unnamed";

    init(newName) {
        this.name = newName;
    }
}

class Dog inherits Animal {
    private breed = "Unknown";

    init(newName, newBreed) {
        this.name = newName;
        this.breed = newBreed;
    }

    bark() {
        print("Woof!");
    }
}

const myDog = Dog("Max", "Golden Retriever");
print(myDog.name);
```

## Native Functions (Standard Library)

The interpreter exposes utility APIs natively within [native-functions.ts](file:///home/hrutav-modha/Documents/programming-language/src/shared/native-functions.ts):

- **I/O Functions**:
  - `print(value)`: Writes `value` to standard output.
  - `input(prompt)`: Displays `prompt` and reads a line from stdin.
- **File System Operations**:
  - `createFile(path)`: Creates an empty file at `path`.
  - `writeFile(path, content)`: Writes `content` to the file at `path`.
  - `deleteFile(path)`: Deletes the file at `path`.
  - `readFile(path)`: Returns the string contents of the file at `path`.
  - `appendFile(path, content)`: Appends `content` to the end of the file at `path`.

## VM Instruction Set

The virtual machine uses a custom instruction set. Each opcode is a 1-byte value. A selection of opcodes is listed below:

| OpCode (Dec) | OpCode (Hex) | Mnemonic | Description |
| :--- | :--- | :--- | :--- |
| **1** | `0x01` | `ADD` | Pops `b` and `a`, pushes `b + a`. |
| **2** | `0x02` | `SUB` | Pops `b` and `a`, pushes `b - a`. |
| **6** | `0x06` | `LOAD_CONST` | Loads a constant from the Constant Pool. |
| **10** | `0x0A` | `DEFINE_VAR` | Defines a variable in the local scope. |
| **11** | `0x0B` | `LOAD_VAR` | Pushes the value of a variable onto the stack. |
| **13** | `0x0D` | `JUMP_IF_FALSE` | Jumps to target if stack top is false. |
| **14** | `0x0E` | `JUMP` | Unconditionally jumps to target. |
| **28** | `0x1C` | `CALL` | Invokes a user-defined function. |
| **30** | `0x1E` | `RETURN` | Returns from a user-defined function. |
| **33** | `0x21` | `DEFINE_CLASS` | Defines a class in the Symbol Table. |
| **34** | `0x22` | `CALL_METHOD` | Invokes a method on an instance. |
| **35** | `0x23` | `GET_PROP` | Retrieves a property value from an instance. |
| **36** | `0x24` | `SET_PROP` | Updates a property on an instance. |
| **37** | `0x25` | `DEFINE_PROP` | Defines a class property with accessibility and staticity modifiers. |

For the complete opcode matrix and stack effects, refer to [OPCODES.md](file:///home/hrutav-modha/Documents/programming-language/OPCODES.md).

## Practical Example

Below is a complete, working example program calculating the Fibonacci sequence using functions, branches, variables, loops, and operations:

```
function fibonacci(n) {
    if n <= 1 {
        return n;
    }
    let prev = 0;
    let curr = 1;
    for let i = 2; i <= n; i = i + 1 {
        let next = prev + curr;
        prev = curr;
        curr = next;
    }
    return curr;
}

// Print the 7th Fibonacci number
print(fibonacci(7)); // Outputs 13
```
