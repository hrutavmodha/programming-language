# Custom Programming Language and Virtual Machine

A lightweight, fully functional custom programming language and virtual machine (VM) written in TypeScript. This project demonstrates the fundamental concepts of compiler construction and virtual machine execution by implementing a complete pipeline: from raw source code to bytecode execution.

## 🚀 Overview

The interpreter processes the source code in several distinct stages:

1. **Lexer**: Reads raw source code and tokenizes it into meaningful keywords, identifiers, literals, and operators.
2. **Parser**: Analyzes the token stream and constructs an Abstract Syntax Tree (AST) representing the program's structure.
3. **Bytecode Generator**: Traverses the AST and compiles it into a linear sequence of low-level instructions (bytecodes/opcodes) for the virtual machine.
4. **Executor (Virtual Machine)**: A stack-based virtual machine that executes the generated bytecodes. It features a constant pool for literals and identifiers, and a scope stack for managing variable environments.

## ✨ Features

- **Variables & Constants**: Declare variables using `let` and constants using `const`.
- **Data Types**: Supports Numbers, Strings, Booleans (`true`, `false`), and `null`.
- **Control Flow**:
  - `if` / `else` statements
  - `switch` statements with `case` and `default` blocks
- **Loops**:
  - `while` loops
  - `do...while` loops
  - `for` loops
  - `break` and `continue` keywords to control loop execution
- **Operators**:
  - Arithmetic: `+`, `-`, `*`, `/`, `%`
  - Comparison: `==`, `!=`, `<`, `>`, `<=`, `>=`
  - Logical: `&` (AND), `|` (OR), `!` (NOT)
- **Built-in Functions**: Easily print output using the `print` keyword.
- **Scoping**: Block-scoped variable management via `ENTER_SCOPE` and `EXIT_SCOPE` opcodes.

## 🛠️ Architecture

- `src/lexer/`: Tokenizes the input string.
- `src/parser/`: Parses tokens into an AST based on the language grammar.
- `src/generator/`: Compiles the AST into bytecodes. Also manages jump points for control flow and loops.
- `src/executor/`: The Virtual Machine. Processes a linear array of bytecodes. Operations are stack-based.
- `src/shared/`: Contains shared resources like the `ConstantPool` (stores strings, identifiers, numbers) and the `ScopeStack` (manages variable environments).

For a detailed list of all supported bytecodes and their stack effects, please refer to the [OPCODES.md](./OPCODES.md) documentation.

## 💻 Example Usage

```typescript
let x = 4;

switch x {
    case 1 {
        print "Hello!";
    } case 2 {
        print "World!";
    } case 3 {
        print "Bye!";
    } case 4 {
        print "How?";
    } default {
        print "Wait!";
    }
}
```

## 🏃 Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/) installed on your machine.
- `npm` (Node Package Manager).

### Installation

1. Clone the repository:
   ```bash
   git clone <repository_url>
   cd <repository_directory>
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

### Running the Interpreter

You can run the existing example defined in `src/index.ts` using a tool like `npx tsx` or `ts-node`:

```bash
npx tsx src/index.ts
```

This will execute the interpreter pipeline and print the tokens, the parsed AST, the generated bytecodes, and finally the output of the virtual machine execution.
