# Parser Module

This module parses a stream of tokens into an Abstract Syntax Tree (AST) using a recursive descent parser.

## 1. Description
The parser consumes the tokens generated by the lexer, matching statements and expressions according to pre-defined grammar rules. It constructs hierarchical node structures reflecting the precedence of arithmetic, logic, comparison, and assignment operations.

## 2. How to Import
```typescript
import Parser from './index.ts'
import ParserState from './state.ts'
```

## 3. Minimal Working Example
```typescript
import Parser from './index.ts'
import ParserState from './state.ts'

// Assumes tokens is an array of Token objects
const state = new ParserState(tokens)
const parser = new Parser(state)

const ast = parser.parse()
console.log(JSON.stringify(ast, null, 2))
```

## 4. API Reference
* [Parser](file:///home/hrutav-modha/Documents/programming-language/src/parser/index.ts): Main class implementing parsing methods.
  * `parse()`: Traverses the token stream and builds the AST representation.
* [ParserState](file:///home/hrutav-modha/Documents/programming-language/src/parser/state.ts): Encapsulates token cursor position and helper methods like `expect()`, `advance()`, and `peek()`.
* AST shapes are defined in [types/nodes.d.ts](file:///home/hrutav-modha/Documents/programming-language/types/nodes.d.ts).

## 5. Design Constraints & Status
* Error recovery is minimal. The parser throws an exception and exits on the first encountered syntax error rather than resuming parsing.
* Expression nesting limits are not statically bounded. Extremely deep AST nests could theoretically trigger call-stack overflow.
